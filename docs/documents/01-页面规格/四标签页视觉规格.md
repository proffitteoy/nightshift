# 四标签页视觉规格（执行版）

更新日期：2026-03-08

## 全局约束
- 移动端优先。
- 浅色、低噪声、可扫读。
- 信息层级优先，不做装饰性设计。

## 页面1：一周热点
### 必备模块
1. 标题与刷新动作
2. 项目卡片列表
3. 点击项目名后进入详情视图，展示两段式项目解读
4. 卡片字段：仓库名、简介、趋势、星标

### 数据字段
- `repo_full_name`
- `description`
- `project_summary`
- `stars_total`
- `trend_7d`

## 页面2：晨报交接
### 必备模块
1. 顶部 URL 输入框
2. 生成区双按钮：
   - `按当前仓库生成`
   - `按 URL 生成`
3. 当前选定仓库状态提示
4. 摘要卡：未解决、高风险、逾期、关键结论
5. 折叠区：
   - `Key Risk Summary`
   - `Handover Records`
   - `项目问答`
6. 明确失败说明文案，用于“最近两次提交不可比较”或“上下文不足”场景

### 数据字段
- `summary.unresolved_count`
- `summary.high_risk_count`
- `summary.overdue_count`
- `summary.key_conclusion`
- `time_range`
- `sections.key_risk_summary`
- `sections.handover_records`
- `question`
- `answer`
- `source`
- `repo_url`

## 页面3：代码全景图
### 必备模块
1. 与其他标签页一致的外层导航和标题栏
2. Android WebView 承载 `/showcase/atlas/`
3. 仓库导入、图谱浏览、节点查看、搜索与 AI 面板全部在工作区内部完成

### 交互约束
- 页面3继续承载 Atlas 工作区，不提供原生轻量替代实现。
- Android Tab3 只负责统一导航与承载，不改写工作区内部交互。
- 若存在订阅列表，工作区可读取首个开启 `code_panorama_enabled` 的仓库作为快捷入口。

## 页面4：订阅中心
### 必备模块
1. 登录、注册、GitHub OAuth 与退出登录
2. 账号信息
3. 订阅列表与 CRUD
4. `Morning Report`、`Code Panorama Updates` 双开关
5. 收件邮箱输入框
6. 发送方式切换：
   - `现在就发送`
   - `选定时间发送`
7. 当前查看仓库状态与“设为当前查看仓库”
8. 运行时配置区：GitHub Token、LLM API Key、Base URL、Model、Timeout、Retries

### 显式移除
- 频率选择 UI
- 更新策略选择 UI
- 调试日志面板

### 数据字段
- 订阅：`repo_url`、`morning_report_enabled`、`code_panorama_enabled`、`recipient_email`、`delivery_mode`、`frequency`、`delivery_time`、`update_strategy`
- 配置：`github_token_configured`、`llm_api_key_configured`、`llm_base_url`、`llm_model`、`llm_timeout_seconds`、`llm_max_retries`
- 当前查看仓库：前端本地状态，不新增后端字段

## 验收要求
- 页面1可从列表进入详情视图，并显示两段式解读。
- 页面2“按当前仓库生成”和“按 URL 生成”语义明确且可用。
- 页面3可进入 `/showcase/atlas/` 工作区并完成核心交互。
- 页面4未登录时只显示账号入口；登录后显示账号信息、订阅与运行时配置。
- 任一接口返回空数据时，对应页面必须给出明确空态提示。
