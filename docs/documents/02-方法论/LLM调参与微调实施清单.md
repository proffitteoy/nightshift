# LLM 调参与微调实施清单

更新日期：2026-03-08

## 1. 目标
- 提升页面2晨报的稳定性、结构性和可执行性。
- 保证模型异常时可回退，页面不阻断。
- 形成可追踪、可回滚、可评估的闭环。

## 2. 当前实现基线
- 入口：`backend/clients/llm_client.py`
- 运行时配置入口：
  - 临时写入：`POST /api/config/llm`
  - 持久化写入：`PUT /api/subscriptions/runtime-config`
- 默认模型：`glm-4-flash`
- 评估数据：`llm_evaluations`
- 关键参数：`model`、`temperature`、`top_p`、`max_tokens`、`timeout_seconds`、`max_retries`

## 3. 实验参数建议
- `temperature`：0.2 / 0.4 / 0.7
- `top_p`：0.8 / 0.9 / 1.0
- `max_tokens`：800 / 1200 / 2000
- `timeout_seconds`：15 / 25 / 40
- `max_retries`：0 / 1 / 2

## 4. 每组实验必须记录
- 样本仓库与时间范围
- 输出快照
- 耗时与失败率
- 结构化字段完整率
- 人工评分

## 5. 提示词治理
- 命名：`场景_模块_版本`
- 新版本必须附：作者、日期、变更原因、回滚版本
- 晨报问答禁止在业务代码中硬编码长提示词
- 晨报主链路优先使用确定性规则，避免时间窗口导致结果漂移

## 6. 上线与回滚
- 新参数先灰度，再全量。
- 任一阈值超线（超时率、结构错误率、人工评分）即回滚至稳定配置。
