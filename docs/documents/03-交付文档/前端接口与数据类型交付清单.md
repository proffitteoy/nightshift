# 前端接口与数据类型交付清单

更新日期：2026-03-08
适用项目：NightShift

## 1. 统一约定

### 1.1 Base URL
- 开发环境：`http://127.0.0.1:8000`
- 当前公网入口：`http://8.163.4.22:8000`
- 页面3静态工作区入口：`/showcase/atlas/`
- 生产建议：使用 HTTPS 域名，不长期使用裸 IP

### 1.2 Content-Type
- 请求：`application/json`
- 响应：`application/json`

### 1.3 Authorization
- Bearer Token：`Authorization: Bearer <access_token>`
- 必须携带 Bearer Token 的接口：
  - `GET /me`
  - `GET/PUT/DELETE /api/subscriptions/runtime-config`
  - `POST /api/subscriptions`
  - `PUT /api/subscriptions/{subscription_id}`
  - `DELETE /api/subscriptions/{subscription_id}`
  - `POST /api/subscriptions/{subscription_id}/send`
- 可选 Bearer Token 的接口：
  - `POST /auth/github/start`
  - `GET /api/subscriptions`
  - `POST /api/project/subscribe`
  - `GET /api/project/daily-report`
  - `POST /api/project/report-by-user`
  - `POST /api/project/report-qa`
  - `POST /api/trending/detail-summary`
  - `GET/POST/OPTIONS /api/proxy`
- 当前语义：
  - 带 Bearer Token 时，后端优先读取当前用户的 GitHub / LLM / 邮件配置与订阅数据。
  - 不带 Bearer Token 时，Tab1 到 Tab3 继续允许匿名使用核心能力；Tab4 写操作仍然拒绝。

### 1.4 统一错误结构
```json
{
  "success": false,
  "error": {
    "code": "FEATURE_DISABLED",
    "message": "feature 'subscriptions' is disabled"
  }
}
```

### 1.5 并发忙碌错误码（HTTP 503）
- `REPORT_BUSY`
- `SUBSCRIBE_BUSY`
- `PANORAMA_BUSY`
- `ANALYSIS_BUSY`

## 2. 账号接口

### 2.1 类型
```ts
export interface AuthRegisterRequest {
  email: string;
  password: string;
  display_name?: string | null;
}

export interface AuthLoginRequest {
  email: string;
  password: string;

export interface UserProfileResponse {
  id: number;

export interface RepoSyncSummary {
  added_count: number;
  skipped_existing_count: number;
  public_repo_count: number;
  private_repo_count: number;

}

export interface AuthTokenResponse {
  access_token: string;
  token_type: "bearer";
  expires_in: number;
  user: UserProfileResponse;
  repo_sync?: RepoSyncSummary | null;
}

export interface GitHubOAuthStartResponse {
  authorize_url: string;
  poll_token: string;
  expires_in: number;

}

export interface GitHubOAuthPollResponse {
  status: "pending" | "completed" | "failed";
  message: string;
  auth?: AuthTokenResponse | null;

```

### 2.2 接口
- `POST /auth/register`
- `POST /auth/login`
- `POST /auth/github/start`
- `GET /auth/github/poll/{poll_token}`
- `GET /auth/github/callback`
- `GET /me`

### 2.3 当前语义
- 密码账号与 GitHub OAuth 账号以 `auth_source` 区分；同邮箱默认不自动合并。
- 已登录状态下调用 `POST /auth/github/start` 表示 connect；未登录状态下表示 OAuth 登录。
- GitHub OAuth 完成后，后端会自动同步当前 GitHub 账号下的公开仓库到当前用户订阅。

- 方法：`POST`
- 路径：`/api/subscriptions/{subscription_id}/send`
- 当前语义：
  - 优先复用当前用户、当前仓库、TTL 内的本地报告。

## 7. 最低联调验收
1. 页面1可展示 `trend_7d` 并通过详情接口显示两段式解读。
2. 页面2可展示 `summary.*` 与 `sections.*`。
3. 页面2在拿不到可比较提交时，必须显示结构化降级结果而不是空白。
4. 页面3可通过 GitHub URL 进入有效工作区。
5. 页面4可完成登录、订阅 CRUD、运行时配置保存与 GitHub OAuth。
6. 页面4未登录时只显示账号入口，登录后才显示受保护区域。
