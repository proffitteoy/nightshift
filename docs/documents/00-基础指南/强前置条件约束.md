# 强前置条件约束

更新日期：2026-03-08

任一条不满足，当前任务判定为 `NO-GO`。

## 1. 范围约束
- 必须写清 `scope_in` 与 `scope_out`。
- 必须标记影响页面（1/2/3/4）。
- 必须关联对应效果图文件。

## 2. 契约约束
- 跨端改动前先冻结接口契约。
- 统一错误结构必须可回归验证。
- 前端接口变化必须同步更新：
  - `android-app/README.md`
  - `android-app/app/README.md`
  - `backend/static/showcase/README.md`
  - `docs/` 相关文档

## 3. 架构约束
- 保留 Android 导航主骨架。
- 路由层不得承载复杂业务逻辑。
- 运行时数据目录不得混入业务代码。

## 4. AI 工程约束
- 提示词必须版本化管理。
- 模型参数必须配置化（含超时、重试）。
- 输出必须经过契约模型校验。
- 密钥不得硬编码入代码与模板配置。

## 5. 安全与发布约束
- 生产环境禁用明文 HTTP。
- 网络超时、错误、空态必须有可见处理。
- 关键链路变更必须附验证证据与回滚方案。
