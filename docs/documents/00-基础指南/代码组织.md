# 代码组织规范（NightShift）

更新日期：2026-03-08

## 1. 顶层结构
```text
NightShift/
├── backend/                # 后端服务（含 Tab3 静态工作区）
├── android-app/            # Android 客户端（保留骨架）
├── docs/                   # 项目文档契约
```

## 2. 运行时目录（按需生成）
- `commit_data/`：仓库活动快照。
- `analysis_data/`：热点分析产物。
- `reports/`：日报产物。
- `sqlite_data/`：SQLite 数据文件。

说明：以上目录属于运行时数据，不属于架构主目录。

## 3. 后端分层约束
```text
backend/
├── api/
├── services/
├── repositories/
├── clients/
├── models/
├── prompts/
└── main.py
```

- `api` 只做协议层工作。
- `services` 负责编排。
- `repositories` 负责持久化。
- `clients` 负责外部调用。
- `models` 负责契约定义。
- `prompts` 负责提示词版本管理。

## 4. 四页面与接口映射
- 页面1（`ui/home`）
  - `GET /api/trending/generate-analysis`
- 页面2（`ui/chat`）
  - `POST /api/project/report-by-user`
  - `GET /api/project/daily-report`
- 页面3（`ui/dashboard`）
  - WebView 直连 `/showcase/atlas/`
  - `GET /api/subscriptions`（订阅预填）
  - `GET /api/proxy`（GitHub 代理克隆）
- 页面4（`ui/notifications`）
  - `GET/POST/PUT/DELETE /api/subscriptions`
  - `GET /api/subscriptions/runtime-config`
  - `PUT /api/subscriptions/runtime-config`

## 5. 前端改造边界
- 不推翻 `MainActivity + Fragment + Navigation`。
- 允许在原页面容器中重构布局和数据绑定。
- 接口契约未冻结前，不做跨端大改。
