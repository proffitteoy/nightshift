## 3. 页面接口模板
### 页面1
- `GET /api/trending/generate-analysis`
- `POST /api/trending/detail-summary`
- 最小字段：`repo_full_name`、`description`、`project_summary`、`stars_total`、`trend_7d`

### 页面2
- `POST /api/project/report-by-user`
- `GET /api/project/daily-report`
- `POST /api/project/report-qa`
- 最小字段：`time_range`、`summary.*`、`sections.*`

### 页面3
- WebView 直连：`/showcase/atlas/`
- 订阅预填：`GET /api/subscriptions`
- GitHub 代理：`GET /api/proxy`

### 页面4
- 订阅管理：
  - `GET /api/subscriptions`
  - `POST /api/subscriptions`
  - `PUT /api/subscriptions/{id}`
  - `DELETE /api/subscriptions/{id}`
- 运行时配置：
  - `GET /api/subscriptions/runtime-config`
  - `PUT /api/subscriptions/runtime-config`
  - `DELETE /api/subscriptions/runtime-config`
- 账号：
  - `POST /auth/register`
  - `POST /auth/login`
  - `POST /auth/github/start`
  - `GET /auth/github/poll/{poll_token}`
  - `GET /me`

## 4. 前端接线模板
- 若已有页面2只调用一个报告接口，必须补 `GET /api/project/daily-report` 与 `POST /api/project/report-qa`。
- 若已有页面4只做订阅 CRUD，必须补 `runtime-config` 的 `GET/PUT/DELETE`。
- 若接入 GitHub OAuth，必须复用后端 callback + poll 闭环，不新增 Android 深链页。

## 5. 发布策略
- 先稳定后端契约，再做 Android 页面绑定。
- 文档与实现必须同轮提交。
