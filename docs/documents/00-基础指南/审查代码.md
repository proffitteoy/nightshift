# 审查代码清单

更新日期：2026-03-05

审查优先级：缺陷 > 回归 > 契约漂移 > 测试缺失 > 文档不同步。

## 一、四页面一致性
1. 页面1是否返回趋势图必须字段（`trend_7d`）。
2. 页面2是否返回 `summary.*` 和 `sections.*`。
3. 页面3是否返回非空 `nodes` 与 `edges`。
4. 页面4是否支持订阅 CRUD 与运行时配置持久化。

## 二、接口契约检查
1. 页面2是否同时支持：
   - `POST /api/project/report-by-user`
   - `GET /api/project/daily-report`
2. 页面4是否已接入：
   - `GET /api/subscriptions/runtime-config`
   - `PUT /api/subscriptions/runtime-config`
3. 错误响应是否统一为 `success/error` 结构。

## 三、后端实现检查
1. 路由层是否仍保持轻量。
2. 外部调用是否有超时、限流、异常处理。
3. 运行时配置是否可持久化并可重启恢复。
4. 关键日志是否包含请求耗时和错误信息。

## 四、AI 工程检查
1. 提示词是否通过注册器管理并版本化。
2. LLM 参数是否可运行时覆盖（包含超时和重试）。
3. 模型失败时是否有规则回退。
4. 评估记录是否写入 `llm_evaluations`。
