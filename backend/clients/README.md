# 外部客户端层说明

更新日期：2026-03-07

## 文件
- `github_client.py`：抓取仓库最近两次提交、compare 结果和近期 PR 活动；同时提供 `fetch_repo_question_context()`，为晨报问答补充 README、根目录结构、近期变更文件、PR 与 Commit 上下文。
- `trending_client.py`：通过 GitHub Search API 抓取热点仓库。
- `llm_client.py`：调用 OpenAI 兼容接口生成晨报问答与页面1详情摘要；失败时规则回退。

## 关键规则
- 客户端层只负责外部调用与数据适配，不做业务编排。
- 所有外部请求必须配置超时、日志和错误处理。
- 不在客户端层保存敏感配置，统一走环境变量和运行时配置接口。

## 当前边界
- 页面2晨报仍以规则摘要为主，但现在会把近期 PR 数据并入报告结构。
- 页面2问答当前优先走 `report_qa_v2`，会结合仓库上下文直接回答用户问题，而不是复述已有摘要。
- 页面1详情摘要支持按项目调用 LLM，并规范为与列表摘要不同的两段式输出；如果未配置模型，服务层负责回退到规则摘要。
- 页面3图谱仍为 MVP 结构化生成，不做真实代码语义解析。
